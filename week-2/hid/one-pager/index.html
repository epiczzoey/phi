문제정의

사용자는 LLM에게 여러 조건이 포함된 작업을 맡기지만, 채팅 UX에서는 조건과 결과가 선형 대화 안에 흩어져 있어, 답변이 조건을 제대로 반영했는지 직접 읽고 검수해야 한다.

즉, 핵심은 **“조건 반영 여부를 사용자가 직접 대조해야 하는 부담”**입니다.

조건 입력 → 답변 생성 → 조건 검수 → 추가 요청이 모두 긴 대화 흐름 안에서 선형적으로 쌓이기 때문입니다.

사용자

  • LLM을 활용하여 여러 조건이 포함된 작업을 요청하는 사용자.
  • 이 사용자는 목적, 형식, 포함할 내용, 제외할 내용 등을 포함해 작업 결과물을 얻는다.
  • LLM과 최소한의 핑퐁으로 만족하는 결과를 얻고자 한다.

페인포인트

  • 누락된 조건이 있으면 다시 추가 요청을 해야 하고, 이때 기존 답변이 바뀌지 않도록 “다른 건 건드리지 말고 이 조건만 추가해줘”처럼 다시 설명해야 한다.
  • 이 과정에서 사용자는 AI에게 일을 맡겼음에도 결과를 계속 검수하고 통제해야 하는 부담을 느낀다.

사용자의 목표

  • LLM에게 여러 조건이 포함된 작업을 요청했을 때, 사용자는 자신이 입력한 목적, 형식, 포함할 내용, 제외할 내용이 빠짐없이 반영된 결과물을 얻고 싶다.
  • 답변 전체를 처음부터 다시 읽지 않아도, 각 조건이 어디에 반영되었는지 빠르게 확인하고 싶다.
  • 누락된 조건이 있을 경우에는 기존 답변 전체를 다시 수정하지 않고, 필요한 부분만 추가하거나 변경하고 싶다.

핵심 마찰 정의

  • 채팅 UX에서는 AI 답변이 긴 메시지로만 나오기 때문에, 사용자는 내가 요청한 조건이 답변 안에 제대로 들어갔는지 직접 찾아야 한다.
  • 채팅 UX는 사용자의 요청 조건과 AI 답변을 연결해서 보여주지 않고 긴 메시지로만 제공하기 때문에, 사용자가 조건 반영 여부를 직접 읽고 대조해야 하는 마찰(불편)이 발생한다.
  • 특히 답변이 길거나 조건이 여러 개일수록, 사용자는 누락된 조건이 있는지 확인하기 위해 전체 답변을 다시 검수해야 한다. 그리고 누락된 내용을 보완할 때도 기존 답변이 바뀌지 않도록 다시 조건을 설명해야 한다.

인터렉션 설계

조건을 구조화하고, 답변 검수 스크롤을 줄이고, 수정 방식까지 분리하는 인터페이스로 확장

하이라이트 or 숏컷 채팅에서는 AI 답변이 긴 메시지로만 나오기 때문에, 사용자는 내가 요청한 조건이 답변 안에 제대로 들어갔는지 직접 찾아야 한다.
하이라이트채팅 UX는 사용자의 요청 조건과 AI 답변을 연결해서 보여주지 않고 긴 메시지로만 제공하기 때문에, 사용자가 조건 반영 여부를 직접 읽고 대조해야 하는 마찰(불편)이 발생한다.
일부 수정과 전체 답변 다시 받기의 옵션을 선택 특히 답변이 길거나 조건이 여러 개일수록, 사용자는 누락된 조건이 있는지 확인하기 위해 전체 답변을 다시 검수해야 한다. 그리고 누락된 내용을 보완할 때도 기존 답변이 바뀌지 않도록 다시 조건을 설명해야 한다.

인터랙션 설계 v2

1. 작업 입력

2. 조건 설계 패널 (PDR 구조)

  • AI 응답은 단순 리스트가 아닌 4가지 카테고리로 구조화된 패널로 노출.
    • 목적
    • 형식
    • 포함할 내용
    • 제외할 내용
  • 각 카테고리의 조건은 체크박스 형태로 기본 체크된 상태로 나열
  • 사용자는 불필요한 조건을 체크 해제
  • 추천 조건
    • 버튼 없이 각 카테고리 하단에 자동으로 노출된다. 칩 형태로 표시되며, 클릭 시 해당 카테고리의 체크리스트에 추가된다. 이미 추가된 항목은 비활성 처리
  • 조건 직접 추가: 각 카테고리에 입력 필드가 항상 존재하며, 텍스트 입력 후 추가 시 해당 카테고리에 체크된 상태로 올라감
  • 하단의 "답변 생성" 버튼을 누르면 현재 체크된 조건 전체를 기준으로 답변을 생성

3. 리뷰 모드 (자동 전환)

  • 답변 생성 완료 시 채팅에서 리뷰 모드로 자동 전환
  • 화면은 좌우 분할 레이아웃으로 구성
    • 왼쪽 — 답변 본문
  • 답변은 섹션 단위(소개 / Step 1~3 / 성공 기준 / 제외 조건 설명)로 나뉘어 표시된다.
    • 조건과 연결된 섹션은 선택 시 하이라이트 처리
  • 답변 하단에는 “+ 조건 추가” 영역을 제공함
    • 카테고리 선택 필드와 조건 텍스트 입력 폼을 표시함
  • 추가한 조건은 우측 조건 맵에 즉시 반영함
  • 오른쪽 고정 패널
    • 조건 검수
      • PDR 카테고리별로 조건을 표시함
      • 각 조건에 상태 배지(반영됨 / 확인 필요)를 표시 함
      • 조건을 클릭하면 답변의 해당 섹션으로 자동 이동하고 하이라이트 처리 됨
    • 섹션 이동
      • 답변의 섹션 목록을 나열
      • 클릭 시 해당 섹션으로 즉시 이동한다. 스크롤 없이 원하는 위치로 이동할 수 있다.

4. 수정 방식 분리

조건 맵에서 조건을 클릭하면 수정 액션 바가 하단에 노출됨

  • 일부 수정
    • 선택한 조건과 연결된 부분만 변경함
    • 모달에서 수정 요청을 입력함
    • 기존 답변은 유지하고, 답변 하단에 “일부 수정 결과” 섹션만 추가함
  • 전체 다시 생성
    • 답변의 방향이나 품질이 전반적으로 맞지 않을 때 사용함
    • 확인 모달을 거친 후, 선택된 조건 전체를 기준으로 새 버전을 답변 하단에 추가함

5. 완료

  • 리뷰 모드 하단의 “완료” 버튼을 누르면 채팅으로 복귀함
  • 채팅 복귀 후 완료 메시지를 표시함
  • 완료 메시지에는 최종 답변 전문을 섹션별로 표시함
  • 완료 이후에도 “답변 수정하기” 버튼을 통해 리뷰 모드에 재진입할 수 있음

테스트 시나리오

  1. 작업 입력
    1. 채팅창에 작업 목적 입력 후 전송
    1. 사용자 버블 생성 확인
  1. 조건 설계
    1. PDR 4카테고리(목적·형식·포함·제외)로 구조화된 조건 패널 노출 확인
    1. 불필요 조건 체크 해제
    1. 누락 조건 직접 입력 후 추가
    1. 추천 조건 선택 → 해당 카테고리에 즉시 반영 확인
  1. 답변 생성 및 리뷰 모드
    1. "답변 생성" 클릭 → 리뷰 모드 자동 전환 확인
    1. 답변이 섹션 단위로 펼쳐진 상태로 노출
    1. 우측 조건 확인 패널: 조건 클릭 → 해당 섹션 하이라이트 확인
    1. 우측 섹션 이동 패널: 항목 클릭 → 해당 섹션으로 즉시 이동 확인
  1. 수정
    1. 조건 선택 후 "일부 수정" 클릭 → 모달 입력 → 기존 답변 유지, 하단에 수정 결과만 추가 확인
    1. "전체 다시 생성" 클릭 → 확인 모달 → 새 버전 섹션 추가 확인
    1. 리뷰 모드 하단 "조건 추가" → 카테고리 선택 + 텍스트 입력 → 우측 패널 즉시 반영 확인
  1. 5단계 · 완료
    1. "완료" 클릭 → 채팅으로 복귀, 완성된 답변 전문 노출 확인
    1. "답변 수정하기" 클릭 → 리뷰 모드 재진입 확인
