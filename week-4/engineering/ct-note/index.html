Retro-Report 초안 생성기

컴퓨팅 사고력 4단계 문서 · PHI 엔지니어링 Track E 최종 과제

마감 2026년 6월 29일 (월) 18:00 주제 프로세스 자동화 기록 프로젝트 A · 프로젝트 B
반복 상황
프로젝트가 끝날 때마다 같은 일이 반복된다. 아웃룩·슬랙·위키를 뒤져 자료를 모으고, 그걸 다시 회고 프레임워크에 맞는 언어로 가공하는 두 단계가 매번 가장 많은 시간을 소비한다.

1 분해 Decomposition

프로젝트 A·B의 retro 작성 과정을 고수준에서 저수준(숫자 단위)까지 하향식으로 분해합니다.

고수준 프로젝트 종료 후, 분산된 문서에서 자료를 수집하고 회고 프레임워크 언어로 가공하여 Retro/Report 초안을 완성한다
중수준 자료 탐색 — 아웃룩·슬랙·위키에서 관련 문서 검색
저수준 탐색 채널: 3개 (아웃룩·슬랙·위키)
저수준 수집 문서: 10~20건 / 프로젝트
저수준 탐색 소요 시간: 30~60분
중수준 내용 분류 — 역할 구분과 회고 정리 (주관적 정보)
저수준 주관 입력 종류: 2종 (내 역할 / 회고)
저수준 종당 입력 분량: ~200자
저수준 회고 항목 수: 7개 (KPT + 4L)
중수준 초안 작성 — 빈 문서에서 섹션별 직접 작성
저수준 주요 출력: 4개 섹션 (SEC1~4) + 보조 출력 3개 (RETRO · DATA · QUESTIONS)
저수준 섹션당 분량: 1~2문단 (~200자)
저수준 작성 소요 시간: 60~90분

4 플로우차트 Algorithm / Flowchart

인터페이스 개요
Glean HTML 파일을 드래그하고 선택적으로 텍스트 2종을 입력한 뒤 버튼을 누르면, AI가 회고 프레임워크에 맞는 7섹션 초안을 카드 형태로 생성·표시하는 단일 페이지 인터페이스

UI 상태

IDLE
파일 미업로드. 드롭존만 표시, "초안 생성" 버튼 비활성.
INPUT
파일 업로드 완료. 텍스트 입력 가능, "초안 생성" 버튼 활성.
LOADING
AI 생성 중. 스피너 표시, 버튼 비활성, 입력 불가.
DONE
4개 섹션 카드 (SEC1~4) 렌더링 완료. 복사·재생성 버튼 활성.
FALLBACK
API 실패 시 자동 전환. 데모 데이터로 4개 섹션 표시, [DEMO] 배지 노출.

프로토타입 구현 계획

순서구현 내용확인 방법
1HTML 파일 업로드 + 텍스트 추출 → gleanDoc 저장콘솔에 gleanDoc 출력 확인
2내 역할·회고 입력창 UI → myRole, reflection 바인딩입력값 콘솔 확인
3프롬프트 조립 + Claude API 연결raw 응답이 콘솔에 출력되는지 확인
4마커 파서 + 4섹션 카드 렌더링 (SEC1~4 주요, RETRO·DATA·QUESTIONS 보조)섹션이 올바르게 분리·출력되는지 확인
5UI 마감 + 복사 버튼 + Vercel 배포전체 플로우 end-to-end 확인
  • 글린 자료만 넣으면 케이스 스터디에 필요한 핵심 맥락을 자동으로 추출한다.
  • 사용자가 긴 입력폼을 채우지 않아도, 부족한 정보만 질문받고 빠르게 초안 생성 단계로 넘어갈 수 있다.
  • KPT·4L 회고 관점으로 프로젝트를 분석해 단순 업무 기록이 아닌 문제 해결 중심의 서사로 재구성한다.
  • 데이터 기반 관점을 반영해 문제 정의, 디자인 판단, 성과 측정 가능성이 드러나는 포트폴리오 문장을 만든다.
  • "뭐부터 써야 하지?"에서 멈추지 않고, 바로 편집 가능한 케이스 스터디 초안을 제공한다.
